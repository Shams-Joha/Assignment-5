Important FAQ

What is Document Object Model (DOM) ?

DOM is a JS Object that represents the page we see in a Web Browser and provides us with an API to interact with it. Our Web browser creates the DOM when it loads an HTML Document & Structures all the documents like a tree. With JS we use DOM to dynamically access any HTML element and modify it to our liking.

How do you select an Element from the DOM ?

We can use several methods for this. For example getElementByID(), getElementByClassName(), getElementByTagName(), querySelector() & querySelectorAll() These methods access the element by it's respective criteria, ex: id, class, tag, selectors etc.

What is Event Delegation in the Context of DOM, Why is it useful?

Event Delegation is the process of efficiently managing & handling events on multiple child elements by attaching a single event listener on a common ancestor. This ancestor listens for events that bubble up from it's child elements. For example, if we have a list of items and want to handle clicks on each one, we can attach a listener to the parent ul instead of each individual li. Inside the event handler, we can use event.target to determine which child element triggered the event & act accordingly. It is useful because it improves performance by reducing event listener count and handles dynamic elements for ex: If elements are added and removed dynamically the parent will still capture events from newly added children.

How do you manipulate an element's attributes & styles using the DOM ?

We can manipulate an element's attributes & styles using various JS methods. For example: We can use getAttribute(), setAttribute() or removeAttribute() to get, set and remove attributes of an element. We can use element.style.__ property to style an element & we also can use methods such as classList.add or remove for dynamically assign classes to an element.
